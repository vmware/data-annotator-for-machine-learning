Frequently Asked Questions:

Q :What types of projects are supported by {{ env.config.serviceTitle }}? A :Text annotation projects where a label should be selected based on a text description and any additional attributes. Tabular annotation projects where a label should be selected based on a large set of attributes. Image annotation projects where allow you to customise the label of the 'Add new label here' input on Labels field. Log annotation projects where allow you to customise the label of the 'Add new label here' input on Labels field. Q : What is the size limit for a {{ env.config.serviceTitle }} dataset? A : Files up to 100MB can be uploaded while creating a new project. Files up to 500MB can be uploaded through the “My Datasets” tab and used to create a new project. If you have GB size datasets, we can support this through a backend script. Please contact {{ env.config.contactEmail }} or {{ env.config.contactSlack }}. Q : What is the format for downloading annotated data? A : Each label type is captured as a new column in the export CSV. Each row contains the number of annotations for a label type. Q : When will you support more other annotation tasks? A : This is on the {{ env.config.serviceTitle }} roadmap. Q : What is active learning and how is it being used? A : Once your project has 50 or more labels, {{ env.config.serviceTitle }} trains an active learning (link to: https://www.datacamp.com/community/tutorials/active-learning ) model on this data. The resulting model runs predictions on the remaining unlabelled set and returns the most uncertain examples for annotators to label. An active learning model continuously queries the "best" and more informative examples for annotators to label without duplicating effort. For example, if there are 100 tweets with a sad face emoji, the model may learn from just a few examples that a sad face emoji " " in your examples is indicative of "negative" sentiment and that it will not benefit from seeing more examples (thus annotators will not need to label it). Rather, it will query for other tweets without sad face emojis.
